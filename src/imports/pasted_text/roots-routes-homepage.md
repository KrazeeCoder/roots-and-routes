Design a visually striking homepage UI for a website called “Roots & Routes: Bothell,” a community resource hub for local residents. The design should feel earthy, grounded, welcoming, and distinctly tied to Bothell’s trails, parks, and community connections. It should look intentional and high-end, not like a generic AI-generated startup landing page.

Core design goal:
Create a homepage that feels memorable, polished, and deeply usable. The page should immediately communicate trust, locality, and clarity. It should feel like a civic resource platform with editorial quality and trail-inspired wayfinding details.

Style direction:
- editorial civic design + trail wayfinding + Pacific Northwest natural textures
- asymmetrical but disciplined composition
- clean grid with strong hierarchy
- refined, grounded, tactile, useful
- subtle organic structure, not whimsical nature branding
- not minimalist to the point of emptiness; not cluttered either

Do NOT use:
- purple or blue gradients
- glassmorphism
- startup SaaS aesthetics
- centered text-only hero sections
- floating abstract blobs
- stock-style vector people
- generic identical feature cards
- excessive roundness
- neon accents
- trendy fintech dashboard visuals

Color palette:
- Canvas #F6F1E7
- Sandstone #E7D9C3
- Sage Dust #A7AE8A
- Olive Trail #6F7553
- Forest Canopy #334233
- Bark #5B473A
- Terracotta #B36A4C
- Clay Shadow #8A6F5A

Color rules:
- mostly warm neutrals and deep greens
- only one warm accent: Terracotta
- no cool blue tones at all
- high contrast for readability
- color should support hierarchy, not decorate randomly

Typography:
- Headings: Fraunces or Cormorant Garamond
- Body/UI: Public Sans or Inter
- Headlines should feel editorial and grounded, not oversized
- Strong hierarchy with clear scale differences
- Body text should be highly legible and practical

Framework constraints:
- Tailwind CSS + shadcn/ui
- 12-column grid
- spacing system should feel intentional and consistent: 24px, 32px, 48px, 80px
- use only 2–3 card styles total
- restrained rounded corners: 12px to 16px
- subtle borders and soft shadows
- hover/focus transitions: 0.2s to 0.3s ease
- all sections must feel connected through repeated motifs and alignment

Homepage structure:

1. Sticky navigation
- standard clear labels: Home, Directory, Spotlights, Events, Suggest a Resource, About
- include one strong CTA button: Explore Directory
- navbar should feel sturdy, calm, and useful

2. Hero section
- left-aligned layout
- headline: “Find your route to local support.”
- supporting text explaining the site helps Bothell residents discover resources, events, and opportunities
- visible directory search bar preview
- filter chips beneath search: Food, Housing, Health, Youth, Jobs, Legal, Events
- right side should include a layered, beautiful visual composition inspired by trail maps, route lines, land contours, and grounded textures
- no generic illustration

3. Quick access waypoints
- 4 to 6 cards for urgent or common needs
- cards should feel like trail markers or resource waypoints
- examples: Food Assistance, Health & Wellness, Housing Support, Youth Programs, Job Help, Community Events

4. Spotlight section
- highlight 3 featured community resources
- one larger featured card and two supporting cards for visual variety
- include category, short summary, who it serves, and CTA
- use Terracotta only as a restrained highlight

5. Find Your Path section
- 3-step visual journey: Search → Filter → Connect
- route-line graphics should subtly reinforce the theme
- should feel elegant and useful, not cartoonish

6. Events preview
- 3 upcoming events or programs
- use a timeline or route-strip layout
- include dates, category tags, and location

7. Suggest a Resource CTA
- warm, community-centered section encouraging users to submit new resources
- include a short sentence about helping Bothell grow stronger together
- visually distinct, but not loud

8. Footer
- complete and structured
- include navigation links, mission statement, reference links, and contact info
- footer should feel designed, not like an afterthought

Visual design requirements:
- use subtle contour lines, trail routes, or map-inspired dividers between sections
- use asymmetry carefully to avoid looking templated
- make one section feel denser and more editorial, another more open and navigational
- create strong visual rhythm between sections
- use texture sparingly: paper, terrain, woodgrain-inspired warmth, or topo-line overlays
- ensure the page feels handcrafted and site-specific

Micro-interactions:
- filter chips should have active/inactive states
- buttons should feel tactile and prominent
- nav links should have subtle animated underline or color transitions
- cards should lift or shift slightly on hover
- search input should feel important and central

Usability priorities:
- extremely easy to scan
- clear calls to action
- obvious information hierarchy
- mobile responsive
- accessible contrast
- no visual gimmicks that reduce clarity

Rubric-oriented goals:
- theme should be obvious and carried throughout the page
- navigation should feel intuitive and efficient
- graphics and color should enhance the user experience
- homepage should preview the required site features: searchable directory, spotlight resources, submission form, and additional useful content
- layout should feel polished enough for competition judging, not like a generic template

Important:
Make the design feel like it was art-directed by a human designer. It should be cohesive, structured, warm, and distinctive — not flashy, not empty, and not obviously AI-generated.
